# 受付GAS セットアップ手順

ブラウザ（公開ページ）から「指示・評価・差し入れ」を受け取り、GitHubに保存する受付係です。
**GitHubトークンはGASの中だけに保管し、ブラウザには絶対に置きません。**

## 1. GASプロジェクトを作る
1. https://script.google.com で新規プロジェクト
2. `Code.gs` の中身を貼り付け

## 2. スクリプトプロパティを設定
「プロジェクトの設定」→「スクリプト プロパティ」に追加：

| キー | 値 |
|---|---|
| `GITHUB_TOKEN` | repo スコープのある Personal Access Token（classic） |
| `REPO_OWNER` | `miyagoshigotono-lgtm` |
| `REPO_NAME` | `game-kaihatubu` |
| `BRANCH` | `main` |
| `PASSPHRASE` | あいことば（ひらがなOK。ページ初回入力と同じ文字列） |

## 3. デプロイ
「デプロイ」→「新しいデプロイ」→ 種類「ウェブアプリ」
- 実行するユーザー：**自分**
- アクセスできるユーザー：**全員**

→ 表示された「ウェブアプリのURL」をコピー。

## 4. ページに接続
`index.html` の先頭付近：

```js
const GAS_ENDPOINT = ''; // ← ここにウェブアプリのURLを貼る
```

URLを貼ってコミットすれば、付箋・評価・差し入れがGitHubに保存されるようになります。
（URLが空のうちは、これまで通りこの端末＝localStorageにだけ保存されます）

## 動作確認
- ブラウザでウェブアプリURLを直接開く → `{"ok":true,"msg":"studio reception is alive"}` が出ればOK。
- ページで付箋を置く → 初回に「あいことば」を聞かれる → 入力後、リポジトリに `instructions.json` が作られる。

## 保存されるファイル
| 種類 | ファイル |
|---|---|
| 指示（付箋） | `instructions.json` |
| 評価 | `ratings.json` |
| 差し入れ | `treats.json` |

いずれも配列に追記され、夜のミーティングがこれを読みます。

## 補足（仕様上の注意）
- ページからGASへの送信は、ブラウザのCORS制約のため **「送りっぱなし（応答は読まない）」** 方式です。
  そのため画面上は楽観的に「送りました」と表示されます。実際に保存されたかはリポジトリの
  `instructions.json` 等で確認できます（合言葉違いは静かに弾かれます）。
